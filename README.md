# rv32i

A RISC-V RV32I processor project — Verilog RTL and a cycle-accurate C++ simulator.

## Repository layout

```
hdl/          Verilog RTL source
tb/           Verilog testbench
sim/          C++ simulator (cache + core)
tests/
  riscv-tests/        official riscv-tests source (git submodule)
  rv32ui-p/           compiled rv32ui-p ELFs (generated by make riscv-tests-build)
  benchmarks/         bare-metal C++ benchmarks (matmul)
scripts/      experiment automation scripts
build/        build artefacts (generated)
results/      experiment output (generated)
```

## Dependencies

| Tool | Purpose |
|---|---|
| `iverilog` / `vvp` | Verilog simulation |
| `g++` (C++17) | C++ simulator build |
| `riscv64-unknown-elf-gcc` | Cross-compile riscv-tests |

Install the cross-compiler on Ubuntu/Debian:
```bash
sudo apt install gcc-riscv64-unknown-elf binutils-riscv64-unknown-elf
```

## C++ Simulator

The simulator is split into two independent components.

### Cache simulator (`sim/cache.h`, `sim/cache.cpp`)

Configurable set-associative cache with multi-level hierarchy support.

**Features**
- Configurable size, block size, associativity, replacement policy (LRU / FIFO / Random), write policy (write-back / write-through), write-allocate
- Multi-level hierarchy (`CacheHierarchy`) — L1 → L2 → … → DRAM
- Dirty eviction propagation between levels
- Miss classification via the 3C model: **cold** (compulsory), **capacity**, **conflict** — using a fully-associative shadow cache updated on every access
- Per-level stats: accesses, hits, misses (by type), evictions (clean + dirty), total cycles, average cycles/access

**Example**
```cpp
CacheConfig l1, l2;
l1.name = "L1"; l1.size = 32768; l1.block_size = 64; l1.ways = 4;
l1.hit_cycles = 4;
l2.name = "L2"; l2.size = 262144; l2.block_size = 64; l2.ways = 8;
l2.hit_cycles = 12;

CacheHierarchy h({l1, l2}, /*memory_cycles=*/200);
uint64_t cycles = h.read(address);
h.print_stats();
```

**Build and test**
```bash
make sim-test       # build + run 63 unit tests (all should pass)
```

### Core simulator (`sim/pipeline.h`, `sim/pipeline.cpp`, `sim/core_sim.cpp`)

Cycle-accurate 5-stage in-order pipeline implementing the full RV32I base ISA.

**Pipeline stages:** IF → ID → EX → MEM → WB

**Features**
- Full forwarding (EX/MEM → EX, MEM/WB → EX) — zero stall cycles for back-to-back ALU instructions
- Load-use hazard detection with 1-cycle stall
- Branch resolution in EX stage with always-not-taken prediction (2-cycle penalty on taken branch)
- Optional branch target buffer (BTB) with 2-bit saturating counters (`--btb`)
- Separate instruction cache (I$) and data cache (D$) — both are `CacheHierarchy` instances; cache miss cycles are added as stall cycles to the total
- ELF32 loader with `tohost`/`fromhost` symbol detection for riscv-tests pass/fail signalling
- Pipeline drain on ECALL/EBREAK/tohost write — in-flight instructions commit before halting so register state is always consistent
- Instruction trace, register dump, and per-level cache statistics on exit

**Default cache configuration**

| Level | Size | Ways | Block | Hit cycles |
|---|---|---|---|---|
| L1-I$ | 32 KiB | 4 | 64 B | 1 |
| L1-D$ | 32 KiB | 4 | 64 B | 4 |
| L2 (shared) | 256 KiB | 8 | 64 B | 12 |
| DRAM | — | — | — | 200 |

**Build**
```bash
make core-sim       # produces build/core_sim
```

**Usage**
```
./build/core_sim [options] <elf>

Options:
  --trace              Print each retired instruction with result
  --trace-pipeline     Print per-cycle pipeline state (IF/ID/EX/MEM)
  --btb [N]            Enable BTB with N entries (default 256, must be power of 2)
  --max-cycles N       Stop after N cycles (default 400M)
  --mem-base ADDR      Memory base address in hex (default 0x80000000)
  --mem-size BYTES     Memory size in bytes (default 67108864 = 64 MiB)
  --l1i-size BYTES     L1 I$ size in bytes (default 32768 = 32 KiB)
  --l1i-ways N         L1 I$ associativity (default 4)
  --l1d-size BYTES     L1 D$ size in bytes (default 32768 = 32 KiB)
  --l1d-ways N         L1 D$ associativity (default 4)
  --l2-size  BYTES     L2 unified size in bytes (default 262144 = 256 KiB; 0 = no L2)
  --l2-ways  N         L2 associativity (default 8)
```

**Example output**
```
$ ./build/core_sim --trace tests/rv32ui-p/rv32ui-p-add

[sim] Loaded tests/rv32ui-p/rv32ui-p-add  entry=0x80000000
[sim] tohost @ 0x80001000
[sim] Starting simulation...

[sim] PASS

=== Core Statistics ===
  Cycles:              1847
  Instructions:        612
  IPC:                 0.331
  Data stall cycles:   8
  I$ stall cycles:     212
  ...
```

**Exit codes:** `0` = pass, positive `N` = riscv-tests failure at test case N, `-1` = memory fault, `-2` = max cycles exceeded.

## riscv-tests integration

The simulator is validated against the official [riscv-tests](https://github.com/riscv-software-src/riscv-tests) RV32UI physical-memory test suite (42 tests covering all RV32I instructions).

**First-time setup** — clone the test suite source:
```bash
git clone https://github.com/riscv-software-src/riscv-tests tests/riscv-tests
cd tests/riscv-tests && git submodule update --init --recursive
```

**Compile the ELFs**
```bash
make riscv-tests-build
```

**Run all tests**
```bash
make riscv-tests
```

Expected output:
```
  PASS  rv32ui-p-add
  PASS  rv32ui-p-addi
  ...
  PASS  rv32ui-p-xori

Results: 42 passed, 0 failed
```

## Benchmarks

A matrix-multiply benchmark (`tests/benchmarks/benchmark.cpp`) exercises the data cache with a realistic access pattern: A rows (stride-1, cache-friendly) × B columns (stride-N, cache-unfriendly). Working set: ~27 KiB (N=48 matrices).

**Build and run with default cache config**
```bash
make bench-run       # compiles and runs tests/benchmarks/matmul
```

**Run cache configuration experiments**
```bash
make bench-experiment   # runs 8 configurations, writes results/matmul.tsv
```

The experiment script (`scripts/cache_experiment.sh`) sweeps three axes:
- **L1 size**: 8 KiB / 32 KiB / 64 KiB
- **L2 size**: 64 KiB / 256 KiB (baseline) / 1 MiB / no L2
- **Associativity**: direct-mapped / 4-way (baseline) / 8-way

It can also be run directly on any ELF:
```bash
./scripts/cache_experiment.sh -o results/out.tsv <elf>
```

The `-o FILE` option writes tab-separated results (config, cycles, instrs, IPC, I$-stalls, D$-stalls, pass/total) alongside the formatted stdout table.
